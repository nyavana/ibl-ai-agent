# Agent Corrections

This file is an approved correction log for durable, repo-local guidance that does not fit cleanly in one skill or reference file.

Do not add entries from feedback alone. Follow the feedback assimilation policy with human approval in `AGENTS.md`: draft the generalized rule, show the target file and wording to the user, and edit this file only after explicit approval.

## How To Use

Before generating or revising reviewed scientific answers, BWM analyses, or repo-local skills, scan this file for approved corrections relevant to the task.

Prefer the most specific applicable instruction:
- `AGENTS.md` for repo-wide runtime or interaction behavior.
- `skills/*/SKILL.md` for workflow-level behavior.
- `skills/*/references/*.md` for technical checks, rubrics, caveats, or examples.
- This file for cross-cutting correction history that does not belong to one skill.

## Entry Template

### YYYY-MM-DD - Short Correction Title

**Status:** Approved by user on YYYY-MM-DD.

**Context:** What task, artifact, or workflow exposed the issue.

**Failure pattern:** The reusable class of mistake to avoid.

**Trigger:** When this correction applies.

**Required behavior:** What future agents must do.

**Forbidden behavior:** What future agents must not do.

**Verification:** How to check compliance before returning an answer.

**Owner:** The durable guidance home, such as `AGENTS.md`, a skill file, a reference file, or this correction log.

**Provenance:** Optional short note linking to the original artifact or discussion.
